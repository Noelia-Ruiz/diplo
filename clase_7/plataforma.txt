5. ¿Por qué te resultaría útil pensar ese objeto como una clase en tu código de automatización?
Pensar ese objeto como una clase permite organizar y agrupar toda la información y los comportamientos relacionados con una clase virtual en un solo lugar.
Con eso, se hace mas facil:
                • Reutilizar el código: para crear muchas clases virtuales diferentes, cambiando solo los valores de sus atributos.
                • Automatizar pruebas de manera sencilla, ya que cada clase virtual puede tener sus propios métodos para simular acciones (como tomar asistencia o mostrar pantalla).
                • Mantener el código ordenado y fácil de entender, ya que cada objeto “sabe” los datos que tiene y qué puede hacer.
                • Modificar o extender funcionalidades de las clases virtuales sin afectar otras partes del sistema.
                • Evitar errores, ya que cada instancia de clase virtual es independiente y no se mezclan los datos entre sí.
